## 简介
数据分析引擎由 [src/background/RuleEventDispatcher.ts](file://src/background/RuleEventDispatcher.ts) 的单例 `dispatcher` 与 `src/background/helper/*` 下的分析器组成。它每秒从 5 秒滑动窗口队列取出事件，计算一个 0–100 的疲劳指数（fatigue index），映射到等级，并在满足条件时触发回调。它是「规则事件分发器」这个文件名背后的真实职责——一个疲劳评估引擎，而非通用事件管线。

## 调度与生命周期
`dispatcher` 通过 `getInstance()` 获取单例。`start()` 以 `TICK_MS = 1000` 注册 `setInterval` 周期调用 `tick()`；`stop()` 清除定时器。外部状态由两个 setter 注入：

- `setVideoFullscreen(active)`：视频全屏态（由 `FullscreenChange` 事件驱动）。
- `setDeviceLocked(locked)`：设备锁屏态（由 `IdleListener` 驱动）。

`onTrigger(cb)` 注册触发回调并返回取消订阅函数；`getLastResult()` / `getWeights()` 暴露只读状态。

## 四项疲劳指标
`computeMetrics()` 从窗口事件计算 4 项原始指标，并各自除以一个「单位」常量做归一化：

| 指标 | 含义 | 来源分析器 | 归一化单位 |
|------|------|-----------|-----------|
| T (tabSwitch) | 标签页切换次数 | `TabSwitchAnalyzer.calculateTabSwitchCount` | `TAB_SWITCH_UNIT = 25` |
| E (mouseEntropy) | 鼠标移动方向熵 | `MouseTrackAnalyzer.calcuateMouseAnthropy` | 本身已归一到 [0,1] |
| D (eyeHandDelay) | 眼手延迟 | `MouseTrackAnalyzer.calculateEyeHandDelay` | `EYE_HAND_DELAY_UNIT = 5` |
| I (eventFrequency) | 事件频率（次/秒） | `EventFrequencyAnalyzer.calculateEventFrequency` | `EVENT_FREQ_UNIT = 10` |

- **鼠标方向熵** 把移动向量方向离散到 `DIRECTION_BINS = 8` 个方向桶，计算香农熵并除以 `log2(8)` 归一到 [0,1]，只统计位移超过 `MIN_DISTANCE_PX = 5` 且时间间隔小于 `MAX_TIME_GAP_MS = 500` 的段。
- **眼手延迟** 取最近一次 `click` 之前、同 url、落点距离点击目标 ≤ `TARGET_RADIUS_PX = 12` 的 mousemove 停留时长（ms）；超过 `MAX_EYE_HAND_DELAY_MS = 2000` 或无匹配则返回 `null`。
- **事件频率** 为窗口事件总数除以 `WINDOW_SECONDS = 5`。

## 休息权重 R
`computeRestWeight()` 依据当前场景返回一个 0–100 的休息权重 `R`，表示"当前状态越像在休息，越应降低疲劳读数"。取所有命中场景的最大值；`deviceLocked` 命中时直接返回其权重：

| 场景 | 常量 | 权重 |
|------|------|------|
| 普通 | `normal` | 0 |
| 视频全屏 | `videoFullscreen` | 30 |
| 鼠标空闲（≥ `MOUSE_IDLE_MS = 20000`） | `mouseIdle` | 40 |
| 窗口失焦（≥ `WINDOW_BLUR_MS = 30000`） | `windowBlur` | 50 |
| 设备锁屏 | `deviceLocked` | 80 |

活动状态由 `updateActivityState()` 依据最近事件时间与窗口焦点维护。

## 融合公式
`tick()` 将归一化指标按可学习权重加权求和得到 `weightedScore`（0–100），再结合休息权重 `R` 与上一次疲劳值的衰减项，得到最终疲劳指数：

```
F = weightedScore × (1 − R / 100) + F_prev × λ
```

其中 `λ = PREV_FATIGUE_DECAY = 0.5` 为上次疲劳的衰减因子，结果被钳制到 `[0, 100]`。四项权重初始各为 `0.25`，由自学习动态调整。

## 疲劳等级与触发去抖
最终疲劳指数按 `THRESHOLD` 映射到 `FatigueLevel`：

| 等级 | 阈值 |
|------|------|
| `mild` | ≥ 60 |
| `moderate` | ≥ 75 |
| `severe` | ≥ 90 |

`maybeFire()` 做触发去抖：仅在**等级升级**，或距上次触发超过 `REFIRE_COOLDOWN_MS = 60000` 时才通过 `onTrigger` 回调对外通知，避免高频重复触发。`service-worker.ts` 目前只是把结果打印到控制台。

## 权重自学习
`recordFeedback(agree)` 依据用户反馈微调对应指标权重（`LEARNING_RATE = 0.05`）：

```
agree:  w += lr × (v / 100) × (1 − w)
reject: w -= lr × (v / 100) × w
```

其中 `v` 为该指标当前贡献值。调整后 `normalizeWeights()` 归一化使权重和为 1。权重通过 `WEIGHTS_STORAGE_KEY = "brainrest_fatigue_weights"` 由 `loadWeights` / `saveWeights` 持久化到 `chrome.storage`。

## 辅助分析器清单
`helper/` 目录下并非所有分析器都接入了 `dispatcher`。下表如实标注接入状态，避免误导：

| 分析器 | 是否被 dispatcher 使用 | 说明 |
|--------|:---:|------|
| `TabSwitchAnalyzer` | 是 | 提供 T 指标 |
| `MouseTrackAnalyzer` | 是 | 提供 E、D 指标 |
| `EventFrequencyAnalyzer` | 是 | 提供 I 指标 |
| `SwitchEntropyAnalyzer` | 否 | 跨类别转移熵（`CATEGORY_COUNT = 11`），仅被 `InteractionMetrics` 引用 |
| `KeyboardAnalyzer` | 否 | 删除键占比，未接入主链路 |
| `InteractionMetrics` | 否 | 聚合 5 指标缩放 0–100，独立实现，未接入 `dispatcher` |
