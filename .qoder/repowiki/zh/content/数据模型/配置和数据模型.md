## 简介

本节说明扩展的配置模型与分类相关数据类型，它们贯穿配置存储、AI 客户端初始化与 URL 分类缓存。

## Option 配置模型

[src/models/Option.ts](file://src/models/Option.ts) 定义扩展的运行配置：

```ts
export interface Option {
  aiProvider: 'openai' | 'deepseek' | AbsoluteUrl;
  categorifyModel: string;
  apiKey: string;
}
```

| 字段              | 类型                                    | 说明                                 |
|-------------------|-----------------------------------------|--------------------------------------|
| `aiProvider`      | `'openai' \| 'deepseek' \| AbsoluteUrl` | AI 服务提供方或自定义 base URL       |
| `categorifyModel` | `string`                                | 分类使用的模型名，默认 `gpt-4o-mini` |
| `apiKey`          | `string`                                | 访问密钥，默认空字符串               |

该模型由 [OptionStore](file://src/services/OptionStore.ts) 读写（存储键 `brainrest_option`，默认值 `openai` /
`gpt-4o-mini` / 空），并被 [AI.ts](file://src/services/AI.ts) 的 `initClient` 消费。

## 基础类型 AbsoluteUrl

[src/models/types.ts](file://src/models/types.ts) 定义模板字面量类型 `AbsoluteUrl`，用于在类型层约束合法的绝对 URL：

```ts
export type AbsoluteUrl = `http://${string}` | `https://${string}`;
```

它作为 `Option.aiProvider` 的自定义 base URL 取值之一，`OptionStore.normalizeProvider` 在运行时以
`startsWith("http://" | "https://")` 做校验。

## UrlCategory 分类枚举

`UrlCategory` 是 URL 分类的联合类型，共 **11** 类：

```ts
export type UrlCategory =
  | 'short_video_entertainment'
  | 'social_feed'
  | 'competitive_progression_games'
  | 'deep_work_productivity'
  | 'longform_deep_reading'
  | 'passive_long_video'
  | 'im_social_adjunct'
  | 'hybrid_learning_cognition'
  | 'low_load_utility'
  | 'shopping_reward_social'
  | 'audio_low_visual';
```

`CategoryService` 会校验模型输出必须属于以上枚举；认知负荷引擎的页面类型基线（`TYPE_BASELINE`）也按这 11 个类别分别映射到 35–90 的负荷分。

## UrlCategoryRecord 存储记录

分类缓存的持久化记录定义在服务层 [UrlCategoryDataBaseManager.ts](file://src/services/UrlCategoryDataBaseManager.ts) 中：

```ts
export interface UrlCategoryRecord {
  domain: string;     // 主键，例如 "example.com"
  category: UrlCategory;
  updatedAt: number;  // 最后更新时间 (ms)
}
```

存入 IndexedDB 数据库 `brainrest_url_categories` 的 `url_categories` 仓库，以 `domain` 为主键，并建立 `domain`（唯一）、
`category`、`updatedAt` 索引。
