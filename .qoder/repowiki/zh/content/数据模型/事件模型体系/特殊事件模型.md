## 简介

除标签页与 UI 事件外，还有三类直接继承自 `Event` 基类的特殊事件：窗口焦点、全屏切换与媒体事件。它们与休息权重 `R` 的判定相关。

## WindowFocus 窗口焦点事件

[src/models/events/WindowFocus.ts](file://src/models/events/WindowFocus.ts)：

```ts
export interface WindowFocus extends Event {
  type: 'focus' | 'blur';
  windowId: number;
}
```

由 [src/background/WindowFocusListener.ts](file://src/background/WindowFocusListener.ts) 产生：
`chrome.windows.onFocusChanged` 回调中，窗口 ID 为 `WINDOW_ID_NONE` 时生成 `blur`，否则生成 `focus`，`url` 为空串并
`queue.push`。窗口长时间失焦（≥ `WINDOW_BLUR_MS = 30000`）会提升休息权重。

## FullscreenChange 全屏切换事件

[src/models/events/FullscreenChange.ts](file://src/models/events/FullscreenChange.ts)：

```ts
export interface FullscreenChange extends Event {
  type: 'fullscreen_change';
  active: boolean;
}
```

由 [src/content/DomListener.ts](file://src/content/DomListener.ts) 监听 `fullscreenchange` 产生，`active`
表示是否进入全屏。视频全屏场景会通过 `engine.setVideoFullscreen` 命中休息场景，提升休息权重 R_rest（`videoFullscreen = 30`）。

**章节来源**

- [src/models/events/FullscreenChange.ts](file://src/models/events/FullscreenChange.ts)
- [src/content/DomListener.ts](file://src/content/DomListener.ts#L1-L129)

export interface MediaEvent extends Event {

> [!NOTE]
