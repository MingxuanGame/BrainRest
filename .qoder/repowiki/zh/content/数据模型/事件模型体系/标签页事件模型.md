## 简介
标签页事件描述浏览器标签页的创建、关闭、URL 变更与激活。它们由背景服务的 `TabListener` 产生并直接入队，是疲劳指标 T（标签页切换次数）的数据来源。

## TabEvent 中间基类
[src/models/events/TabEvent.ts](file://src/models/events/TabEvent.ts) 在 `Event` 基础上增加 `tabId`：

```ts
export interface TabEvent extends Event {
  tabId: number;
}
```

## 具体标签页事件
| 事件 | `type` | 额外字段 | 说明 |
|------|--------|----------|------|
| `TabCreated` | `tab_created` | 无 | 新标签页被创建 |
| `TabClosed` | `tab_closed` | 无 | 标签页被关闭 |
| `TabChanged` | `tab_changed` | `old_url`, `new_url` | 标签页 URL 发生变化 |
| `TabActivated` | `tab_activated` | `windowId` | 标签页被激活（切换到前台） |

`TabCreated` 与 `TabClosed` 仅声明各自的 `type` 字面量；`TabChanged` 记录变更前后的 URL；`TabActivated` 记录所属窗口 ID。

## 由谁产生
[src/background/TabListener.ts](file://src/background/TabListener.ts) 用 `tabUrls: Map<number, string>` 维护每个标签页的当前 URL，并映射 Chrome API：

- `chrome.tabs.onCreated` → `tab_created`
- `chrome.tabs.onRemoved` → `tab_closed`
- `chrome.tabs.onUpdated`（仅 `changeInfo.url` 存在时）→ `tab_changed`（填充 `old_url` / `new_url`）
- `chrome.tabs.onActivated` → `tab_activated`（填充 `windowId`）

这些事件通过 `queue.push` 进入 5 秒滑动窗口，供 `TabSwitchAnalyzer` 统计切换次数。
