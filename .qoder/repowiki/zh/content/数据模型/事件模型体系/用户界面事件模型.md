## 简介
用户界面事件描述页面内的鼠标、键盘、滚动与触摸交互。它们由内容脚本的 `DomListener` 采集，是鼠标方向熵、眼手延迟、事件频率等疲劳指标的原始输入。

## UiEvent 中间基类
[src/models/events/UiEvent.ts](file://src/models/events/UiEvent.ts) 在 `Event` 基础上增加可选的目标元素信息：

```ts
export interface UiEvent extends Event {
  targetTag?: string;
  targetId?: string;
  targetClass?: string;
}
```

## 具体 UI 事件
| 事件 | `type` | 额外字段 |
|------|--------|----------|
| `UiClickEvent` | `click` | `button`, `clientX`, `clientY` |
| `UiKeyEvent` | `keydown` \| `keyup` | `key`, `code`, `altKey`, `ctrlKey`, `shiftKey`, `metaKey` |
| `UiMouseMoveEvent` | `mousemove` | `clientX`, `clientY` |
| `UiScrollEvent` | `scroll` | `scrollX`, `scrollY` |
| `UiTouchEvent` | `touchstart` \| `touchend` \| `touchmove` | `identifier`, `clientX`, `clientY`, `force`, `radiusX`, `radiusY`, `rotationAngle` |

`UiKeyEvent` 与 `UiTouchEvent` 的 `type` 是多个字面量的联合，用以在同一模型内区分子动作。

## 由谁产生
[src/content/DomListener.ts](file://src/content/DomListener.ts) 注册对应的 DOM 监听器：

- `mousemove` 以 `MOUSE_MOVE_SAMPLE_MS = 200` 采样，生成 `UiMouseMoveEvent`；
- `click` 生成 `UiClickEvent`；
- `keydown` / `keyup` 生成 `UiKeyEvent`；
- `scroll` 生成 `UiScrollEvent`；
- `touchstart` / `touchend` / `touchmove` 遍历 `changedTouches`，为每个触点生成一条 `UiTouchEvent`。

每条事件的 `url` 取 `window.location.href`，并尽量填充目标元素的 `targetTag` / `targetId` / `targetClass`，最后经 `sendEvent` 通过 Port 送往背景服务。
