## 简介
本指南面向 BrainRest 贡献者。BrainRest 是一个 Chrome MV3 扩展，通过内容脚本采集用户交互事件，在后台 service worker 中以滑动窗口计算“疲劳指数（fatigue index）”，并在超过阈值时触发提醒（当前为控制台输出）。此外后台还提供基于 AI 的 URL 分类能力。本文只描述仓库中真实存在的实现。

## 环境搭建
- Node.js：建议使用较新的 LTS 版本（Vite 8 要求较新 Node）。
- 安装依赖：在项目根目录运行 `npm install`。
- 开发：`npm run dev` 启动 Vite（CRXJS 提供扩展 HMR）。
- 构建：`npm run build`（`tsc -b && vite build`），产物在 `dist`。
- 加载扩展：Chrome 打开 `chrome://extensions`，开启开发者模式，「加载已解压的扩展程序」，选择 `dist` 目录。
- 代码检查：`npm run lint`。

## 项目结构
按功能域组织于 `src` 下：
- `background/`：service worker 及其监听器、事件队列、疲劳指数引擎（`RuleEventDispatcher`）与 `helper/` 分析器。
- `content/`：注入页面的内容脚本，采集 DOM 事件并通过 Port 上报，另含 `AutoCategorizer` 触发分类。
- `models/`：事件模型（`models/events`）、`Option`、`types` 等类型定义。
- `services/`：AI 客户端、分类服务、Option 存储、两个 IndexedDB 管理器。
- `popup/`：React 占位 UI（当前未在清单中启用）。
- `manifest.ts`：MV3 清单源。

## 运行时架构
存在两条独立通路：
1. **事件流（长连接 Port）**：内容脚本通过 `chrome.runtime.connect({ name: "event-stream" })` 建立 Port，`DomListener` 采集的 UI 事件经 `EventChannel.sendEvent` 发送；service worker 在 `onConnect` 中接收并 `queue.push(event)`。后台的 `TabListener`/`WindowFocusListener`/`IdleListener` 直接向同一队列推送标签页/窗口/空闲事件。
2. **分类请求（一次性消息）**：`AutoCategorizer` 在页面加载后经 `chrome.runtime.sendMessage` 发送 `{ type: "categorize", url, html }`；service worker 用 `isCategorizeRequest` 校验后调用 `getCategory` 返回 `{ ok, domain, category }`。

## 关键模块职责
- **service-worker.ts**：导入监听器（副作用注册），`dispatcher.start()`，注册 `onTrigger` 打印疲劳结果；处理 Port 连接与 categorize 消息。
- **RuleEventDispatcher.ts**：单例 `dispatcher`，每秒 `tick` 从 `queue` 读取 5s 窗口事件，计算 4 个归一化指标（tabSwitch/mouseEntropy/eyeHandDelay/eventFrequency），加权求和后按公式 `F = weightedScore × (1 − R/100) + F_prev × λ` 融合休息权重 R 与上次疲劳衰减，分级（60/75/90），并支持基于用户反馈的权重自学习（持久化到 `chrome.storage.local`）。
- **EventQueue.ts**：`SlidingWindowQueue`，`push()` 时按 5000ms 窗口 `trim()`，`getEvents()` 返回窗口内事件副本。
- **helper/**：`TabSwitchAnalyzer`、`MouseTrackAnalyzer`（鼠标方向熵与眼手延迟）、`EventFrequencyAnalyzer` 被 dispatcher 使用；`SwitchEntropyAnalyzer`、`KeyboardAnalyzer`、`InteractionMetrics` 目前未接入 dispatcher。
- **content**：`DomListener` 采集 mousemove（采样）、click、keydown/keyup、scroll、touch、fullscreenchange；`EventChannel` 负责 Port 发送；`AutoCategorizer` 发起分类。
- **services**：`AI`（OpenAI/DeepSeek 客户端）、`CategoryService`（11 类分类 + DB 缓存）、`OptionStore`（配置存取）、`UrlCategoryDataBaseManager`（域名分类库）、`EventDataBaseManager`（事件库，**当前未被引用**）。

## 新增功能指引
- **新增一种采集事件**：在 `models/events` 中定义事件接口（继承 `Event`/`UiEvent`/`TabEvent`），在 `DomListener` 或对应后台监听器中通过 `createEvent` 构造并发送/入队。
- **让疲劳指数使用新指标**：在 `helper/` 添加分析器，在 `RuleEventDispatcher.computeMetrics` 中接入并扩展 `FatigueMetrics`/`FatigueWeights`（注意初始权重需归一化）。
- **消费疲劳结果**：`dispatcher.onTrigger(cb)` 注册回调（返回取消订阅函数），或 `dispatcher.getLastResult()` 读取最近结果。
- **新增消息类型**：在 `messages.ts` 定义请求/响应类型与类型守卫，并在 service worker 的 `onMessage` 中处理。

## 调试技巧
- **后台**：在 `chrome://extensions` 中打开该扩展的 service worker「检查视图」，可看到 `dispatcher` 打印的疲劳结果与 categorize 处理日志。构建默认不压缩且带 sourcemap，便于断点。
- **内容脚本**：在目标页面 DevTools 的 Sources 中定位 `src/content/*`，观察事件采集与 Port 发送。
- **存储**：疲劳权重存于 `chrome.storage.local` 键 `brainrest_fatigue_weights`；Option 存于 `brainrest_option`；分类结果存于 IndexedDB `brainrest_url_categories`。

## 代码规范
- TypeScript 严格选项已开启（`noUnusedLocals`、`noUnusedParameters` 等），提交前确保 `tsc -b` 与 `npm run lint` 通过。
- 错误处理采用原生 `throw new Error(...)` 的轻量方式，无统一日志系统。
- 事件对象统一经 `createEvent` 创建，保证 `timestamp`/`type`/`processed`/`url` 字段一致。
