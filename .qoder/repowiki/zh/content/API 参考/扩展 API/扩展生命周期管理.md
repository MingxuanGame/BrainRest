## 简介
BrainRest 在 Manifest V3 下以 service worker 承载后台逻辑。生命周期围绕 worker 的加载、监听器注册与调度器启停展开。

## service worker 启动
worker 由 `src/manifest.ts` 声明为 ES module。加载时 [service-worker.ts](file://src/background/service-worker.ts) 以副作用方式导入三个监听器模块，随后调用 `dispatcher.start()` 并注册 `onTrigger` 回调与两类运行时监听（`onConnect` / `onMessage`）。

> [!NOTE]
> MV3 service worker 可能在空闲时被浏览器回收。当前代码在模块顶层直接注册监听器与启动定时器，worker 被唤醒重新加载时会重新执行这些注册；文档不臆造额外的持久化/复活逻辑。

## 监听器注册时机
`import "./TabListener"` 等语句在模块求值阶段即执行 `chrome.*.addListener`，因此监听器随 worker 加载而注册，无需显式初始化函数。

## 调度器生命周期
[RuleEventDispatcher](file://src/background/RuleEventDispatcher.ts) 提供：
- `start()`：以 `TICK_MS = 1000` 注册 `setInterval` 周期 `tick`；
- `stop()`：清除定时器；
- `onTrigger(cb)`：注册触发回调，返回取消订阅函数。

`service-worker.ts` 仅调用 `start()`，未主动 `stop()`。

## 空闲状态检测
[IdleListener](file://src/background/IdleListener.ts) 调用 `chrome.idle.setDetectionInterval(60)`，在 `onStateChanged` 时把 `state === "locked"` 注入 `dispatcher.setDeviceLocked`；启动时还 `queryState` 做一次初始对齐。锁屏会显著提升休息权重（`deviceLocked = 80`）。
