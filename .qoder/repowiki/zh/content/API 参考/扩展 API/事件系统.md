## 简介

事件系统指内容脚本与背景服务之间用于传输采集事件的长连接通道。与分类请求走 `sendMessage` 不同，事件流走持久的 `Port`
，以支持高频事件的连续投递。

## 长连接 Port

[src/content/EventChannel.ts](file://src/content/EventChannel.ts) 在内容脚本加载时建立连接并导出发送函数：

```ts
const port = chrome.runtime.connect({ name: "event-stream" });

export function sendEvent(event: Event): void {
  port.postMessage(event);
}
```

连接名固定为 `"event-stream"`，`DomListener` 采集到的每条事件都通过 `sendEvent` → `port.postMessage` 发送。

## 背景端接收

[src/background/service-worker.ts](file://src/background/service-worker.ts) 在 `chrome.runtime.onConnect` 中按名称过滤：

```ts
chrome.runtime.onConnect.addListener((port) => {
  if (port.name !== "event-stream") return;
  port.onMessage.addListener((event) => {
    queue.push(event);
  });
});
```

收到的事件被压入 5 秒滑动窗口队列 [EventQueue](file://src/background/EventQueue.ts)，供认知负荷引擎每 30 秒取用；`page_complexity` 事件不入队，直接转发引擎。

## 与一次性消息的区别

| 维度      | 事件流（Port）              | 分类请求（sendMessage） |
|-----------|-----------------------------|-------------------------|
| 连接方式  | 长连接 `connect`            | 一次性 `sendMessage`    |
| 名称/标识 | `"event-stream"`            | `type: "categorize"`    |
| 方向      | 内容脚本 → 背景（单向投递） | 请求-响应               |
| 用途      | 高频 DOM/交互事件           | 页面分类                |
