## 简介
BrainRest 使用两种持久化机制：`chrome.storage.local`（键值型，用于配置与权重）和 IndexedDB（经 `idb` 库封装，用于 URL 分类缓存）。

## chrome.storage.local
- **配置存储**：[OptionStore](file://src/services/OptionStore.ts) 以键 `brainrest_option` 读写 `Option`，见 [配置 API](../配置%20API.md)。
- **疲劳权重**：[RuleEventDispatcher](file://src/background/RuleEventDispatcher.ts) 以键 `brainrest_fatigue_weights`（`WEIGHTS_STORAGE_KEY`）通过 `loadWeights` / `saveWeights` 持久化自学习后的四项权重。

`chrome.storage.local` 在 MV3 后台与 popup 中均可用，是替代 `localStorage` 的统一存储 API。

## IndexedDB（idb）
[UrlCategoryDataBaseManager](file://src/services/UrlCategoryDataBaseManager.ts) 单例 `urlCategoryDB` 管理数据库 `brainrest_url_categories`（v1），仓库 `url_categories`，主键 `domain`，索引 `domain`（唯一）/`category`/`updatedAt`。提供 `put`/`batchPut`/`getExact`/`listByCategory`/`lookup`/`lookupCategory`/`delete`/`clear`。

> [!NOTE]
> 另有 [EventDataBaseManager](file://src/services/EventDataBaseManager.ts)（数据库 `brainrest`，仓库 `events`）已实现但当前未被任何模块引用。

## 存储用途汇总
| 存储 | 键 / 数据库 | 用途 | 使用方 |
|------|-------------|------|--------|
| `chrome.storage.local` | `brainrest_option` | 扩展配置 | OptionStore |
| `chrome.storage.local` | `brainrest_fatigue_weights` | 疲劳权重 | RuleEventDispatcher |
| IndexedDB | `brainrest_url_categories` | URL 分类缓存 | UrlCategoryDataBaseManager |
| IndexedDB | `brainrest`（events） | 事件持久化（未使用） | EventDataBaseManager |
