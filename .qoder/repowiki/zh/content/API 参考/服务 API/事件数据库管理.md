## 简介
[src/services/EventDataBaseManager.ts](file://src/services/EventDataBaseManager.ts) 以单例 `eventDB` 封装事件的 IndexedDB 持久化能力，基于 `idb`。

## 数据库结构
| 项 | 值 |
|------|------|
| 数据库名 | `brainrest` |
| 版本 | 1 |
| 对象仓库 | `events` |
| 主键 | `timestamp` |
| 索引 | `processed` |
| 保留窗口 | `DAY_MS`（24 小时） |

## 方法一览
| 方法 | 说明 |
|------|------|
| `batchPut` | 批量写入事件，并清理 24 小时前的旧数据 |
| `getUnprocessed` | 通过 `processed` 索引取未处理（值为 0）的事件 |
| `updateByTimestamp` | 按主键更新单条 |
| `markProcessed` | 将事件标记为已处理 |
| `prune` | 清理过期事件 |

## 实现状态说明
> [!IMPORTANT]
> `EventDataBaseManager` 已完整实现，但当前代码库中**没有任何模块 import 或调用 `eventDB`**。疲劳引擎直接消费内存中的 5 秒滑动窗口队列（`SlidingWindowQueue`），事件并不落盘 IndexedDB。因此 `processed` 索引、`markProcessed` 等能力在当前实现里没有实际使用路径。本文如实标注，避免将该管理器误述为主数据链路的一部分。
