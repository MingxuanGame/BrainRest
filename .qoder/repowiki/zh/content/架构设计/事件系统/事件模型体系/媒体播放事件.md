## 简介

本节描述与媒体相关的事件模型。 **重要说明：`MediaEvent` 已定义但当前代码从未构造或发射它**，属于预留模型。与视频相关、真正被使用的是
`FullscreenChange`（全屏切换）。

## MediaEvent 接口

```ts
export interface MediaEvent extends Event {
  type: 'media';
  tab_id: number;    // 播放媒体所在标签页
  paused: boolean;   // 当前是否暂停
  url: string;       // 页面或媒体源 URL
}
```

## 实现状态

- ❌ 没有任何监听器发射 `media` 事件；`DomListener` 与后台监听器均未涉及媒体播放/暂停。
- 因此疲劳引擎当前不会收到 `MediaEvent`。若未来要感知视频播放状态，需要新增内容脚本对 `<video>`/`<audio>` 的 `play`/
  `pause` 监听并经事件通道上报。

## 相关：全屏切换

真正用于“视频全屏”场景判断的是 `FullscreenChange`：

```ts
export interface FullscreenChange extends Event {
  type: 'fullscreen_change';
  active: boolean;   // 是否有元素处于全屏
}
```

`DomListener` 监听 `fullscreenchange` 发射该事件；后台可据此设置 `engine.setVideoFullscreen(...)`，命中“视频全屏”休息场景。
