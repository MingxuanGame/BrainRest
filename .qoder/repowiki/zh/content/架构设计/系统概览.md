## 系统定位

BrainRest（"Train your brain to rest better!"）是一个 Chrome MV3
扩展，通过监测用户在浏览器中的交互行为，实时估算"脑休息指数 BRI"（Brain Rest Index），并在满足触发条件时输出触发路径供前端决策休息提醒。同时它能对访问的网站做 AI
分类，为认知负荷分析提供上下文。

## 能力清单

- **事件采集**：鼠标移动/点击、键盘、滚动、触摸、全屏切换（内容脚本）；标签页创建/关闭/切换/激活、窗口焦点、系统空闲（后台）；页面复杂度采样（每 30s）。
- **脑休息指数 BRI**：每 30s 基于认知负荷（CL_cog）与身体疲劳（CL_phy）双通道融合计算，输出 0–100 分与分级（low/moderate/high）。
- **触发路径评估**：硬门槛 + 三条触发路径（A 持续高负荷 / B AUC 累积 / C 神经肌肉疲劳），命中结果附在 BRIResult.triggerPath 中。
- **URL 分类**：调用 OpenAI/DeepSeek 兼容接口把域名归入 11 类，并用 IndexedDB 缓存。
- **配置管理**：AI Provider / 模型 / API Key 存于 `chrome.storage.local`。
- **个人校准**：k_personal 系数在线学习（0.5–1.5），每 7 天自动校准。

## 权限与宿主

`manifest.ts` 声明 `permissions: ["tabs", "windows", "storage", "idle"]`，`host_permissions` 覆盖 `api.openai.com` 与
`api.deepseek.com`（供后台调用 AI）。内容脚本注入范围为 `<all_urls>`，后台为 module 类型 service worker。

## 模块地图

- `content/`：`index`、`DomListener`、`EventChannel`、`AutoCategorizer`、`PageComplexityAnalyzer`
- `background/`：`service-worker`、`EventQueue`、`TabListener`、`WindowFocusListener`、`IdleListener`、`helper/*`
- `background/engine/`：`CognitiveLoadEngine`、`CognitiveLoadCalculator`、`PhysicalFatigueCalculator`、`TriggerEngine`、`SessionTracker`、`DataQualityGate`、`BRIHistoryBuffer`、`TabEventBuffer`、`PersonalCalibration`、`types`
- `services/`：`AI`、`CategoryService`、`OptionStore`、`UrlCategoryDataBaseManager`、`EventDataBaseManager`（未接入）
- `models/`：`events/*`、`Option`、`types`、`TimeData`（未接入）
- `popup/`：`App`、`main`（占位，未启用）
