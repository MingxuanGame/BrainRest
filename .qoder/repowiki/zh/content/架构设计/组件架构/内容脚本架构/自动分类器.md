## 简介

`AutoCategorizer` 是内容脚本中负责“页面自动分类”的模块。它自身不做分类，只把当前页面的 URL 与 HTML 中转给后台，由后台完成分类并落库。

## 触发时机

在 `window` 的 `load` 事件后再延迟 3 秒执行：

```ts
window.addEventListener("load", () => {
  setTimeout(() => { void autoCategorize(); }, 3000);
});
```

延迟是为了等待页面 UI 稳定后再抓取 HTML，提高分类准召率。

## 请求内容

构造 `CategorizeRequest` 并通过 `chrome.runtime.sendMessage` 发送：

```ts
const request: CategorizeRequest = {
  type: "categorize",
  url: window.location.href,
  html: document.documentElement.outerHTML,
};
await chrome.runtime.sendMessage(request);
```

## 为何在后台执行分类

分类需要 `apiKey`，而它只应保存在 service worker（页面上下文不可访问）。因此内容脚本仅中转数据，后台 `getCategory` 完成 AI
调用与 IndexedDB 落库。分类结果不进入事件流，避免污染 UI 事件采集。

## 错误处理

`sendMessage` 被 `try/catch` 包裹，失败时静默忽略——分类失败不应影响页面正常的事件采集。当前实现是“即发即忘”，不读取
`sendResponse` 返回值。
