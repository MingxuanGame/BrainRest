- TypeScript 使用 Project References 将浏览器端（tsconfig.app.json）与 Node 端（tsconfig.node.json）类型检查分离，顶层 tsconfig.json 仅聚合引用。
- 所有 tsconfig 统一启用 verbatimModuleSyntax、moduleDetection: force、erasableSyntaxOnly、noUnusedLocals、noUnusedParameters 等严格现代 TS 选项。
- ESLint 采用 Flat Config 格式，通过 defineConfig([...]) 数组形式组合 js、typescript-eslint、react-hooks、react-refresh 规则集。
- 构建脚本遵循 tsc -b 先行类型检查、再 vite build 打包的顺序，确保类型错误在构建阶段即被拦截。
